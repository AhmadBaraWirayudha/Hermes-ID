# UI Style Guide

The interface uses a modern flat design system with no emojis or emoticons.

## Requirements applied

- Black background.
- Earthy color palette.
- No emoji or emoticon text in the UI codebase.
- No em dash characters in text files.
- Minimal animation only: subtle fade-in and small hover elevation.

## Palette

| Token | Hex | Usage |
|---|---|---|
| black | `#050505` | app background |
| panel | `#0B0B0A` | cards and containers |
| panel-soft | `#11100E` | metrics and alerts |
| earth | `#C2A878` | primary accent |
| sage | `#8A9A5B` | secondary accent |
| clay | `#A66A4C` | warm accent |
| sand | `#D8C3A5` | highlighted text |
| muted | `#A99B83` | secondary text |
| border | `#2A241D` | dividers and outlines |

## Design notes

- Avoid gradients except very subtle panel depth.
- Avoid shadows; use borders and spacing.
- Keep interactions minimal and fast.
- Prefer concise labels and clean tables.
- Use Plotly dark charts with the same earth palette.
