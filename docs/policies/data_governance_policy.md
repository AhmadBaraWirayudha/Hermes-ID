# Data Governance Policy

## Purpose

Ensure market data, source data, scraped data, imported CSV files, and generated reports are accurate, traceable, lawful, and fit for use.

## Rules

1. Every source must have an owner.
2. Every source must pass source onboarding review.
3. Raw data must be retained when legally permitted for auditability.
4. Processed data must preserve source, date, item, region, metric, and currency fields.
5. Data quality checks must be reviewed before external reporting.
6. Licensed market data must not be redistributed unless permitted by contract.
7. Personal data must not be mixed into market observations unless approved.

## Evidence

- source registry
- scrape logs
- data catalog
- processed file manifests
- data quality reports
